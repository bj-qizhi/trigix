# Triage Labels

The project uses the default canonical triage role names as label strings.

## Category labels

| Role | Label |
|---|---|
| bug | `bug` |
| enhancement | `enhancement` |

Every triaged issue should have exactly one category label.

## State labels

| Role | Label |
|---|---|
| needs-triage | `needs-triage` |
| needs-info | `needs-info` |
| ready-for-agent | `ready-for-agent` |
| ready-for-human | `ready-for-human` |
| wontfix | `wontfix` |

Every triaged issue should have exactly one state label.

## State meanings

- `needs-triage`: maintainer needs to evaluate.
- `needs-info`: waiting on reporter or product owner for more information.
- `ready-for-agent`: fully specified and ready for an AFK agent.
- `ready-for-human`: needs human implementation or judgment.
- `wontfix`: will not be actioned.
